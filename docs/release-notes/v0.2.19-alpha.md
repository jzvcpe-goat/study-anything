# v0.2.19-alpha

Study Anything v0.2.19-alpha turns external material import into a productized, verifiable path for
platform Agents and local Skills.

## Added

- Added `learning-context-package-v1`, a stable import schema for platform-collected `web`,
  `document`, `video_slice`, `app_context`, `markdown_note`, and `obsidian_note` excerpts.
- Added `POST /v1/context-packages/validate`, `POST /v1/sessions/from-context-package`, and
  `POST /v1/sessions/{session_id}/context-package`.
- Added CLI commands `context-validate` and `context-import`.
- Added bundled importer plugin examples:
  - `plugins/example-web-importer`
  - `plugins/example-note-importer`
- Added NotebookLM-style fixtures under `fixtures/notebooklm/` and the importer release gate
  `scripts/verify_importer_lesson_flow.py`.
- Expanded platform packs and generated OpenAPI/OpenAI tools with Learning Context Package import
  tools.

## Changed

- Obsidian export now has safer frontmatter quoting, stable Study Anything tags, vault-safe filenames,
  and preserved Obsidian backlinks from imported context metadata.
- Skill Mode demo now verifies the importer-based lesson flow in addition to the existing Skill,
  platform, gateway, and eval checks.
- Plugin manifests now support `read:context` and `write:context` permissions for importer workflows.

## Security And Privacy

- Study Anything still does not store real model credentials or external Agent secrets.
- Learning Context Packages may contain bounded private excerpts at the import boundary. Validation
  rejects secret-like keys and common API-key/token patterns.
- Obsidian and `learning-package-v1` exports continue to omit raw source and enrichment text.
- NotebookLM-style support is fixture-based and manual/export-oriented; it does not depend on or call
  an official NotebookLM API.

## Known Limitations

- Importer plugins are discovered and validated through manifests, but plugin entrypoints are not
  executed by the runtime yet.
- NotebookLM-style bridge support is a package boundary and fixture flow, not a live NotebookLM sync.
- LanceDB retrieval and richer importer SDK execution remain future work.
