# v0.2.6-alpha

Study Anything v0.2.6-alpha is the recommended self-host release for MVP trials that need stronger
launch readiness evidence, safer recovery guidance, and a more coherent workbench UI.

## Highlights

- Adds read-only recovery readiness surfaces:
  - `GET /v1/recovery/status`
  - recovery status embedded in `GET /v1/system/status`
  - Web Launch panel coverage for backup, restore safeguards, and privacy warnings
- Adds `scripts/verify_backup_restore_drill.py`, a disposable Docker drill that:
  - creates an isolated Compose project
  - verifies the API and Web proxy smoke before backup
  - mutates state
  - restores from backup
  - confirms session count rollback
  - removes test containers and volumes on success
- Improves Docker source-build reliability for non-ASCII checkout paths by running the drill from an
  ASCII temp source copy when needed.
- Hardens local plugin registry trust:
  - bundled plugin source digests
  - registry digest verification in plugin discovery and preview
  - Ed25519 registry-signature verification when trusted keys are configured
- Expands the Web same-origin smoke to verify:
  - learning flow
  - recovery status
  - encrypted Sync export and inspect
  - plugin registry trust
  - local PMF metrics privacy
- Refines the Web UI toward the Image2 notebook/workbench concept:
  - Learn remains natural-language first
  - Agent setup is arranged as a focused connection workbench
  - Launch surfaces PMF, runtime, Sync, recovery, and interest capture without losing the local-first trust boundary
- Retains the Bring Your Own Agent learning loop, Postgres state, optional FalkorDB projection,
  LangGraph workflow, local encrypted Sync package, local PMF signals, plugin trust summaries, Skill
  Mode, and multi-architecture GHCR images.

## First Launch

Install Docker Desktop or Docker Engine with the Compose plugin, start the daemon, then run:

```bash
python3 scripts/setup_env.py
USE_PUBLISHED_IMAGES=true ./scripts/launch_self_host.sh
```

Open http://localhost:5173.

If you want to build from source instead of using published images, keep the checkout path ASCII-only:

```bash
git clone https://github.com/jzvcpe-goat/study-anything.git ~/study-anything
cd ~/study-anything
python3 scripts/setup_env.py
./scripts/launch_self_host.sh
```

After launch, verify the public API flow and Web-origin smoke:

```bash
API_BASE=http://127.0.0.1:8000 python3 scripts/verify_full_api_flow.py
WEB_BASE=http://127.0.0.1:5173 python3 scripts/verify_full_stack_web.py
```

## Upgrade

Create a local backup before upgrading:

```bash
python3 scripts/self_host_data.py backup
```

Then pull the release images:

```bash
STUDY_ANYTHING_IMAGE_TAG=v0.2.6-alpha \
USE_PUBLISHED_IMAGES=true \
./scripts/launch_self_host.sh
```

For a full local recovery rehearsal, run:

```bash
python3 scripts/verify_backup_restore_drill.py
```

The drill uses disposable Docker resources and cleans them up on success.

## Known Limitations

- The alpha remains self-host-first. Managed hosting, accounts, billing, Publish, and Teams are not
  included yet.
- Study Sync is not a hosted service yet. This release includes the local encrypted package boundary,
  not remote storage, cross-device conflict resolution, account recovery, or support tooling.
- Recovery is intentionally manual. The Web/API surfaces show status and commands but do not execute
  destructive restore operations.
- Plugin installation remains local and explicit. Remote marketplace review, signed update
  distribution, and payment boundaries are not implemented yet.
- Workspace roles are local metadata, not a hosted account or enterprise identity provider.
- First-run GHCR downloads can take several minutes on slower connections.
- LanceDB retrieval and Playwright browser automation remain planned for a later alpha.
