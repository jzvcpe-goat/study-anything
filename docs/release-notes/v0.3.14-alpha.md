# Study Anything v0.3.14-alpha

This alpha hardens the Learning Enrichment operator bridge for platform-agent,
NotebookLM-style, Obsidian, and second-brain workflows. It keeps Study Anything
as a local-first learning engine, not a standalone frontend.

## Added

- `scripts/verify_learning_enrichment_bridge.py` emits
  `learning-enrichment-bridge-verification-v1`.
- The bridge verifier proves coverage for `web`, `document`, `video_slice`,
  `app_context`, `markdown_note`, and `obsidian_note` context.
- The verifier checks the redacted Markdown+HTML micro-lesson, NotebookLM
  manual bridge metadata, Obsidian export, strict second-brain handoff, local
  archive manifest, source hashes, and platform pack evidence.

## Changed

- Platform packs for Kimi, Codex, and WorkBuddy now require the Learning
  Enrichment bridge verifier and acceptance evidence.
- Ecosystem submission, operator drill, manual rehearsal, bundle manifest,
  adoption pack, and release check now include the bridge gate.
- Docs now describe the Learning Enrichment Layer as the platform Agent bridge
  for external context and local memory handoff.

## Security And Privacy

- Real model keys remain outside Study Anything.
- Raw source text, raw enrichment text, browser/video/app private context,
  Agent endpoint secrets, and real model credentials are excluded from bridge
  evidence.
- Direct user-owned Obsidian and learning-package exports may include answers
  selected by the learner; strict second-brain handoff excludes learner answers
  and grading feedback.

## Verification

```bash
python3 scripts/verify_learning_enrichment_bridge.py --check
python3 scripts/verify_ecosystem_submission_pack.py
python3 scripts/generate_platform_adoption_pack.py --check
./scripts/release_check.sh
```

## Known Limits

- This release does not add a standalone UI.
- NotebookLM support remains a manual bridge contract, not an official
  NotebookLM API integration.
- External browsing, application control, video slicing, and real model/tool
  execution stay in the user's platform Agent or local HTTP Agent.
