# v0.2.28-alpha

Study Anything v0.2.28-alpha hardens clean deployment, first-run onboarding, and platform-agent
diagnostics. This release keeps the launch path API/Skill Mode first and does not require the
standalone frontend.

## Added

- `GET /v1/deployment/guide` returns `deployment-guide-v1`: redacted Skill Mode, Docker source-build,
  and GHCR published-image launch commands for Kimi, Codex, WorkBuddy, and local operators.
- `scripts/diagnose_adoption.py` now emits `adoption-diagnostics-v1` with
  `adoption-diagnostic-plan-v1`, including copyable next commands for missing `.env`, Docker
  unavailable, API unreachable, Agent endpoint unavailable, missing provider defaults, and GHCR
  manifest checks.
- `scripts/verify_published_image_launch.py --allow-pull-timeout-report` now includes manifest
  evidence and explicit fallback acceptance criteria when local GHCR layer pulls are too slow.
- The platform tool manifest includes `study_anything_deployment_guide`, so HTTP-tool platforms can
  query launch guidance after the local API is reachable.
- The platform adoption pack now includes `doctor.sh`, `launch_self_host.sh`, `stop_self_host.sh`,
  and the published-image verifier.

## Changed

- Release defaults now point to `v0.2.28-alpha`.
- Self-host, adoption, GitHub launch, roadmap, and platform-pack docs explain the deployment guide
  and diagnostic plan.
- Platform tool generation, adoption pack generation, operator drill, and external adoption checks
  include the deployment guide contract.

## Privacy And Security

- Study Anything still does not store real model API keys. Real reasoning stays inside the user's
  platform Agent or user-owned HTTP Agent gateway.
- Deployment guidance is metadata-only. It must not return raw source text, learner answers, Agent
  endpoint secrets, or model credentials.
- Published-image fallback evidence is only acceptable with a successful GitHub `docker-images`
  workflow and a manifest containing both `linux/amd64` and `linux/arm64`.

## Verify

```bash
python3 scripts/setup_env.py
./scripts/doctor.sh
python3 scripts/diagnose_adoption.py
API_BASE=http://127.0.0.1:8000 python3 scripts/verify_platform_agent_tools.py
python3 scripts/verify_published_image_launch.py --tag v0.2.28-alpha --pull-timeout-seconds 180 --allow-pull-timeout-report
./scripts/release_check.sh
```

## Known Limits

- GHCR layer pulls can still be slow on some local networks. Use the timeout report plus GitHub
  workflow and manifest evidence when reviewing release readiness.
- Hosted Sync, Teams, Publish, billing, and plugin marketplace payments remain future commercial
  surfaces, not part of this local-first alpha.
- Standalone frontend polish is intentionally out of scope for this release.
