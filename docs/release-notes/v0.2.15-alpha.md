# v0.2.15-alpha

Study Anything v0.2.15-alpha makes the platform-Agent-first path easier to verify from a clean clone.
The release adds a dry-run mode to the same OpenAI-compatible gateway used for Kimi and similar
providers, then wires that gateway into platform packs, docs, bundle assets, and local acceptance
scripts.

## Highlights

- Adds `AGENT_GATEWAY_MODE=dry_run` support to `scripts/openai_compatible_agent_gateway.py`.
- Adds `scripts/verify_openai_compatible_gateway.py` for:
  - gateway-only contract checks without a real key
  - API registration as a user-owned HTTP Agent provider
  - teaching layers, quiz, grading, mastery, `agent-audit`, and `agent-eval/artifact`
  - privacy checks that redacted audit/eval artifacts do not expose source text, answers, endpoint,
    or model secret names
- Extends the OpenAI-compatible gateway prompt and dry-run output for `teach.overview`,
  `teach.glossary`, `teach.examples`, `note.scribe`, `source.verify`, and `embedding.create`.
- Updates `agent-add-http --set-default` so a new HTTP Agent is registered for the full learning
  capability set by default.
- Updates Kimi, Codex, and WorkBuddy platform packs with copy-ready dry-run acceptance commands.
- Updates the platform bundle manifest to include the gateway script, gateway verifier, and gateway
  acceptance commands.
- Updates default published-image tags to `v0.2.15-alpha`.

## First Launch

Use Skill Mode when you want the fastest local verification:

```bash
./scripts/run_skill_mode_demo.sh
```

For Docker self-host:

```bash
python3 scripts/setup_env.py
USE_PUBLISHED_IMAGES=true ./scripts/launch_self_host.sh
```

Maintainers can verify the public API image after GHCR publish:

```bash
python3 scripts/verify_published_image_launch.py --tag v0.2.15-alpha
```

## Platform Agent Flow

Verify the OpenAI-compatible gateway without credentials:

```bash
python3 scripts/verify_openai_compatible_gateway.py --gateway-only
```

With a running local API, verify the full gateway-backed learning loop:

```bash
API_BASE=http://127.0.0.1:8000 python3 scripts/verify_openai_compatible_gateway.py
```

Then switch the gateway to a real provider such as Kimi by setting:

```bash
export AGENT_LLM_BASE_URL="https://api.moonshot.cn/v1"
export AGENT_LLM_API_KEY="$MOONSHOT_API_KEY"
export AGENT_LLM_MODEL="${AGENT_LLM_MODEL:-kimi-k2.6}"
```

Study Anything still does not store real model keys. The user's gateway or platform Agent owns model
choice, credentials, tools, browsing, and external data.

## Known Limitations

- The alpha remains API/Skill/CLI/platform-agent-first. There is no shippable standalone Web UI.
- The dry-run gateway proves contract and wiring, not real model answer quality.
- Browser-only Kimi chat still cannot operate local scripts or call `127.0.0.1`; use the local
  gateway, a terminal-capable Agent, or an authenticated private gateway.
- Managed hosting, billing, remote Sync, Teams, and marketplace payments are not included yet.
- External eval adapters are packaging targets; native release gates still use redacted deterministic
  artifact checks.
