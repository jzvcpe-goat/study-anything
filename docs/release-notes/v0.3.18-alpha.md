# Study Anything v0.3.18-alpha

This release turns field-adoption failures into a GitHub-first support desk loop for external
platform users and maintainers.

## Highlights

- Added `platform-support-triage-v1`, a redacted support triage report for external platform
  adoption failures.
- Added `platform-support-issue-template-v1` GitHub issue templates for platform import failures,
  local gateway failures, published-image pull failures, Agent eval evidence failures, and docs
  confusion.
- Added `platform-support-ticket-fixture-v1` mock support tickets that include version, platform,
  command, diagnostic code, fixture id, redacted logs, and next commands.
- Added a maintainer playbook for schema mismatch, missing local gateway, unsupported auth mode, tool
  naming drift, timeout, localhost restrictions, package corruption, and version drift.
- Wired support triage into release checks, ecosystem submission metadata, platform packs, adoption
  pack generation, bundle manifest, docs, and tests.

## Verification

- `python3 scripts/generate_platform_support_triage.py --check`
- `python3 scripts/verify_platform_support_triage.py --check`
- `python3 scripts/verify_platform_support_triage.py --pack platform/generated/study-anything-platform-adoption-pack.zip`
- `./scripts/release_check.sh`

## Known Limits

- The support tickets are deterministic mock fixtures, not live GitHub issues from external users.
- The support desk is manual and GitHub-first. There is no hosted support portal, automatic support
  upload, or managed account workflow in this release.
