# Study Anything v0.3.19-alpha

This alpha turns the v0.3.18 support desk into a first-adopter onboarding and maintainer readiness
gate.

## Added

- `platform-onboarding-readiness-v1` generated report.
- `first-external-adopter-walkthrough-v1` for Kimi, Codex, WorkBuddy, and generic OpenAPI/MCP paths.
- `maintainer-sla-labels-v1` and `maintainer-rotation-checklist-v1`.
- `platform-triage-dashboard-v1` JSON and Markdown dashboard.
- `platform-release-blocker-fixture-v1` mock release-blocker fixtures.
- `docs/adopter-onboarding.md` and `docs/maintainer-rotation.md`.

## Verification

```bash
python3 scripts/generate_platform_onboarding_readiness.py --check
python3 scripts/verify_platform_onboarding_readiness.py --check
python3 scripts/verify_platform_onboarding_readiness.py \
  --pack platform/generated/study-anything-platform-adoption-pack.zip
./scripts/release_check.sh
```

## Boundaries

- No standalone frontend is required.
- No real model keys are stored by Study Anything.
- No raw source text, learner answers, Agent prompts, Agent endpoints, browser/video/app private
  context, or personal profile data are included in shared onboarding evidence.
- Hosted Sync, Publish, Teams, billing, SSO, and marketplace payments remain post-PMF work.
