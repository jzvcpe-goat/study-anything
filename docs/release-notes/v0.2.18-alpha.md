# v0.2.18-alpha

Study Anything v0.2.18-alpha moves the project further into a Skill/Plugin-first shape: platform
agents can run one enriched lesson, collect quality evidence, and export user-owned knowledge
artifacts without opening a standalone Web UI.

## Highlights

- Added `learning-package-v1`, a portable JSON export for platform-agent handoff, NotebookLM-style
  bridge workflows, Obsidian pipelines, and local archives.
- Added `GET /v1/sessions/{session_id}/exports/learning-package`.
- Expanded the CLI with `enrich`, `teach`, `quality-eval`, `obsidian-export`, `package-export`, and
  `lesson` commands.
- Added `scripts/verify_platform_lesson_flow.py`, an end-to-end platform-Agent smoke that completes
  one enriched lesson and verifies Agent audit, Agent eval, quality eval, Obsidian export, and the
  learning package.
- Updated Kimi, Codex, and WorkBuddy packs to require the enriched lesson verifier and
  `learning-package-v1` evidence.
- Regenerated OpenAPI, OpenAI-compatible tool definitions, the tool catalog, and the platform bundle
  manifest.

## Privacy Boundary

- Study Anything still does not store model API keys.
- The learning package includes source references and excerpt hashes, not raw source or enrichment
  text.
- Obsidian and learning-package exports are user-controlled private learning data. Platform wrappers
  should not log generated teaching content, learner answers, grading feedback, or insights by
  default.

## Known Limits

- The NotebookLM bridge is still a package/export boundary, not an official NotebookLM API adapter.
- The independent Web UI remains out of scope for this release.
- Real teaching quality still depends on the user-owned Agent behind the gateway; native quality gates
  are minimum release gates, not a full pedagogy benchmark.
