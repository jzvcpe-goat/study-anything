# v0.2.5-alpha

Study Anything v0.2.5-alpha is the recommended self-host release for MVP trials that need a stronger
local-first trust boundary before any hosted service exists.

## Highlights

- Adds local plugin trust summaries for explicit local installs:
  - source digest
  - review metadata status
  - signature metadata status
  - risk level
  - install recommendation
- Adds `GET /v1/plugins/trust-policy` so operators and plugin authors can inspect the alpha plugin
  trust boundary from the public API.
- Adds local workspace ownership:
  - hashed local identities
  - default workspace assignment
  - `owner`, `admin`, `member`, and `viewer` roles
  - session `workspace_id`
- Adds the encrypted local sync package foundation for future Study Sync:
  - `GET /v1/sync/status`
  - `POST /v1/sync/export`
  - `POST /v1/sync/inspect`
- Keeps Sync local-first in this release. Study Anything does not store the package passphrase, upload
  packages, create hosted accounts, or enable billing.
- Extends the full API smoke script to verify encrypted package export, inspect, and plaintext leakage
  boundaries.
- Retains the Bring Your Own Agent learning loop, Postgres state, optional FalkorDB projection,
  LangGraph workflow, local PMF signals, backup/restore tooling, Skill Mode, and multi-architecture
  GHCR images.

## First Launch

Install Docker Desktop or Docker Engine with the Compose plugin, start the daemon, then run:

```bash
python3 scripts/setup_env.py
USE_PUBLISHED_IMAGES=true ./scripts/launch_self_host.sh
```

Open http://localhost:5173.

If you want to build from source instead of using published images, keep the checkout path ASCII-only:

```bash
git clone https://github.com/jzvcpe-goat/study-anything.git ~/study-anything
cd ~/study-anything
python3 scripts/setup_env.py
./scripts/launch_self_host.sh
```

After launch, verify the public API flow:

```bash
API_BASE=http://127.0.0.1:8000 python3 scripts/verify_full_api_flow.py
```

## Upgrade

Create a local backup before upgrading:

```bash
python3 scripts/self_host_data.py backup
```

Then pull the release images:

```bash
STUDY_ANYTHING_IMAGE_TAG=v0.2.5-alpha \
USE_PUBLISHED_IMAGES=true \
./scripts/launch_self_host.sh
```

## Encrypted Sync Package

Create a local encrypted package with a passphrase you control:

```bash
curl -X POST http://localhost:8000/v1/sync/export \
  -H 'Content-Type: application/json' \
  -d '{"passphrase":"choose a long local passphrase"}'
```

Inspect a package without restoring plaintext:

```bash
curl -X POST http://localhost:8000/v1/sync/inspect \
  -H 'Content-Type: application/json' \
  -d '{"passphrase":"choose a long local passphrase","package":{...}}'
```

The package envelope excludes source text, answers, raw user IDs, Agent endpoints, Agent metadata, and
plugin source code in plaintext. The encrypted payload can contain local learning state and local
configuration needed for future restore workflows.

## Known Limitations

- The alpha remains self-host-first. Managed hosting, accounts, billing, Publish, and Teams are not
  included yet.
- Study Sync is not a hosted service yet. This release includes the local encrypted package boundary,
  not remote storage, cross-device conflict resolution, account recovery, or support tooling.
- Plugin installation is local and explicit. Remote marketplace review, signing, updates, and payment
  boundaries are not implemented yet.
- Workspace roles are local metadata, not a hosted account or enterprise identity provider.
- First-run GHCR downloads can take several minutes on slower connections.
- The Web Launch panel is functional but still visually dense in desktop layouts.
- LanceDB retrieval and Playwright browser automation remain planned for a later alpha.
