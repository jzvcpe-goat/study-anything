# v0.2.27-alpha

Study Anything v0.2.27-alpha hardens the plugin ecosystem so platform Agents
can inspect, validate, and route local plugins before any install or runtime
execution. The release keeps the product local-first and plugin-friendly while
preserving the rule that real model credentials stay in the user's Agent or
platform runtime.

## Added

- `GET /v1/plugins/sdk` returns `plugin-sdk-v1`, the typed hook contract for
  importers, enrichment builders, exporters, source verifiers, Agent tools,
  Agent panels, and Agent provider templates.
- `GET /v1/plugins/capabilities` returns `plugin-capability-index-v1` with
  installed plugin hooks, declared/inferred capabilities, permission risks,
  trust reports, and alpha runtime notes.
- `POST /v1/plugins/validate-package` returns
  `plugin-package-validation-v1` for one local plugin directory without copying
  files or executing entrypoints.
- Plugin manifests now support `schemaVersion: plugin-manifest-v1`,
  `description`, and `capabilities`.
- Added `plugins/example-enrichment-importer`, a deterministic importer plus
  enrichment artifact template.
- Extended `plugins/example-exporter` with a second-brain handoff export
  function and `export.second_brain_handoff` capability.
- Added `docs/plugin-sdk.md` and `docs/plugin-registry.md`.

## Changed

- Platform tools now include plugin SDK, capability inventory, and package
  validation tools for Kimi, Codex, WorkBuddy-style workspaces, and generic
  HTTP tool platforms.
- Platform packs and adoption-pack generation now include Plugin SDK and
  registry docs.
- Release smoke checks now validate plugin SDK privacy invariants:
  no remote code downloads, no package copy during validation, no entrypoint
  execution, no plugin source code, no learner answers, and no Agent secrets.

## Not Included

- No hosted plugin marketplace.
- No automatic remote plugin downloads or updates.
- No plugin payment flow.
- No browser or model-key access inside Study Anything plugins.
