# v0.2.0-alpha

Study Anything v0.2.0-alpha is the first recommended self-host release for real local trials.

## Highlights

- Bring Your Own Agent HTTP contract with fake deterministic demo support and a user-owned
  OpenAI-compatible gateway example for providers such as Kimi.
- Durable app Postgres session state with compiled LangGraph workflow execution and Postgres
  checkpointing.
- Optional privacy-preserving Langfuse observations and FalkorDB topology projection.
- Idempotent FalkorDB topology rebuild APIs with Postgres remaining the canonical store.
- Skill Mode bootstrap and CLI flow for terminal-capable agents.
- Explicit local plugin installer with manifest validation and overwrite protection.
- Natural-language-first bilingual Web UI for learning and Agent setup.
- Core Docker profile for first-run self-hosting, with optional full observability services.
- Published GHCR API and Web images for both `linux/amd64` and `linux/arm64`.
- Local backup and destructive restore tooling with SHA-256 verification, private environment
  snapshots, and CI coverage.
- Node.js 24-compatible GitHub Actions, retry-hardened Compose smoke, React 19, and Vite 8.

## Upgrade

Create a local backup before upgrading:

```bash
python3 scripts/self_host_data.py backup
```

Use the published images:

```bash
STUDY_ANYTHING_API_IMAGE=ghcr.io/jzvcpe-goat/study-anything/api:v0.2.0-alpha \
STUDY_ANYTHING_WEB_IMAGE=ghcr.io/jzvcpe-goat/study-anything/web:v0.2.0-alpha \
docker compose \
  --env-file .env \
  -f infra/compose/docker-compose.yml \
  -f infra/compose/docker-compose.images.yml \
  up -d
```

## Known Limitations

- The alpha is self-host-first. Managed hosting, accounts, billing, Sync, Publish, and Teams are not
  included yet.
- Agent provider defaults are stored in the local `study_anything_data` Docker volume.
- Plugin installation is local and explicit. Remote marketplace review, signing, and install UI are
  not implemented yet.
- FalkorDB and Langfuse are optional operational services. They are not canonical learning stores.
- LanceDB retrieval and Playwright browser automation are planned for the next alpha.
