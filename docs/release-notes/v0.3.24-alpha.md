# Study Anything v0.3.24-alpha

This alpha hardens published-image launch verification for operators running
from slow or cache-only Docker environments.

## Highlights

- `scripts/verify_published_image_launch.py --skip-pull` now uses cached-only
  Docker Compose semantics instead of allowing an implicit unbounded pull.
- Compose startup is bounded by `--compose-up-timeout-seconds` and can return a
  JSON diagnostic when paired with `--allow-pull-timeout-report`.
- New manifest-only verification reports distribution readiness without claiming
  runtime smoke success.
- Published-image evidence now covers cached image misses, Compose startup
  timeouts, and manifest-only runtime-unverified handoff states.

## Validation

```bash
python3 scripts/verify_published_image_launch.py --tag v0.3.24-alpha --manifest-only
python3 scripts/generate_published_image_evidence.py --check
python3 scripts/verify_published_image_evidence.py --check
python3 scripts/verify_release_asset_adoption.py --fixture fixtures/release-asset-adoption/asset-only-pass.json --asset-dir platform/generated --runtime metadata-only
./scripts/release_check.sh
```

## Known Limits

- Manifest-only mode proves that the release image is published for required
  platforms. It does not replace a successful container smoke on a runner with
  reliable Docker/GHCR access.
