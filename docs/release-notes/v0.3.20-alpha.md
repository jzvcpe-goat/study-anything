# Study Anything v0.3.20-alpha

This alpha publishes the maintainer-facing support status layer on top of the
v0.3.19 onboarding readiness gate.

## Added

- `public-support-status-v1` generated public status report.
- `public-maintainer-dashboard-v1` JSON and Markdown dashboard.
- `public-status-linkage-fixture-v1` fixtures for every maintainer SLA label.
- `docs/public-support-status.md`.
- Release-gate, adoption-pack, ecosystem-submission, operator-drill, manual
  rehearsal, and external-adoption proof wiring for public support status.

## Verification

```bash
python3 scripts/generate_platform_public_support_status.py --check
python3 scripts/verify_platform_public_support_status.py --check
python3 scripts/verify_platform_public_support_status.py \
  --pack platform/generated/study-anything-platform-adoption-pack.zip
./scripts/release_check.sh
```

## Boundaries

- The public dashboard is metadata-only.
- No standalone frontend is required.
- No real model keys are stored by Study Anything.
- No raw source text, learner answers, Agent prompts, Agent endpoints,
  browser/video/app private context, full support bundle payloads, or personal
  profile data are included in public status evidence.
- Hosted Sync, Publish, Teams, billing, SSO, and marketplace payments remain
  post-PMF work.
