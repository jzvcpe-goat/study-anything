# Study Anything v0.3.17-alpha

This release turns platform adoption diagnostics into a reusable field rehearsal path for Kimi,
Codex, WorkBuddy-style HTTP workspaces, and generic OpenAPI platforms.

## Highlights

- Added `platform-field-adoption-rehearsal-v1`, a redacted rehearsal report with platform-specific
  import transcripts, an import quirks catalog, and support handoff evidence.
- Added `platform-import-failure-fixture-v1` mock fixtures for schema mismatch, missing local
  gateway, unsupported auth mode, tool naming drift, timeout, browser localhost restrictions,
  package corruption, and version drift.
- Wired field rehearsal evidence into platform packs, ecosystem submission metadata, adoption pack
  generation, bundle manifest, operator drill, manual rehearsal, external adoption proof, release
  checks, and docs.
- Kept all field evidence local-first and redacted: no raw source text, learner answers, Agent
  prompts, real Agent endpoints, model keys, judge keys, or browser/video private context.

## Verification

- `python3 scripts/generate_platform_field_rehearsal.py --check`
- `python3 scripts/verify_platform_field_rehearsal.py --check`
- `python3 scripts/verify_platform_field_rehearsal.py --pack platform/generated/study-anything-platform-adoption-pack.zip`
- `./scripts/release_check.sh`

## Known Limits

- Field rehearsal transcripts are deterministic redacted handoff artifacts, not live Kimi,
  WorkBuddy, or marketplace submission logs.
- Real platform import behavior still depends on the host platform's OpenAPI/tool importer and
  localhost/private-network policy.
