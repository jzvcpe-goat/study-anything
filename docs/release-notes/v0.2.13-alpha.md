# v0.2.13-alpha

Study Anything v0.2.13-alpha publishes the post-`v0.2.12-alpha` platform-ecosystem hardening work.
It makes the API/Skill-first product easier to package for Codex, Kimi-compatible agents,
WorkBuddy-style workspaces, and private Agent platforms without turning Study Anything into a
model-key manager or standalone frontend.

## Highlights

- Adds Agent Eval release gates:
  - `scripts/verify_agent_eval_assets.py`
  - `scripts/run_external_agent_evals.py`
  - CI and release-check validation for redacted `agent-eval-artifact-v1` adapter assets.
- Adds copy-ready platform packs:
  - `platform/packs/codex`
  - `platform/packs/kimi`
  - `platform/packs/workbuddy`
- Adds `scripts/verify_platform_ecosystem_packs.py` so the packs stay aligned with the source
  platform manifest, privacy contract, and required Agent audit/eval acceptance evidence.
- Adds `platform/generated/study-anything-platform-bundle.json`, a deterministic bundle manifest
  with sha256 hashes for platform packs, generated import assets, key docs, and the repo-local Skill
  entrypoint.
- Adds `scripts/generate_platform_bundle_manifest.py --check` and wires it into CI and release checks.
- Updates default published-image tags to `v0.2.13-alpha`.

## First Launch

Install Docker Desktop or Docker Engine with the Compose plugin, start the daemon, then run:

```bash
python3 scripts/setup_env.py
USE_PUBLISHED_IMAGES=true ./scripts/launch_self_host.sh
```

Open:

- API docs: http://localhost:8000/docs
- API health: http://localhost:8000/v1/health
- System status: http://localhost:8000/v1/system/status

Maintainers can verify the public API image end to end with:

```bash
python3 scripts/verify_published_image_launch.py --tag v0.2.13-alpha
```

## Platform Agent Assets

Use the source manifest and generated import assets when building an integration:

```text
platform/study-anything-platform-tools.json
platform/generated/study-anything-platform-openapi.json
platform/generated/study-anything-openai-tools.json
platform/generated/study-anything-tool-catalog.md
platform/generated/study-anything-platform-bundle.json
```

Use the platform packs as copy-ready starting points:

```text
platform/packs/codex
platform/packs/kimi
platform/packs/workbuddy
```

Validate a running integration with:

```bash
API_BASE=http://127.0.0.1:8000 python3 scripts/verify_platform_agent_tools.py
API_BASE=http://127.0.0.1:8000 python3 scripts/verify_agent_eval_flow.py
```

The platform tool surface intentionally excludes Agent provider setup, plugin installation, encrypted
sync export, PMF export, and other management APIs.

## Upgrade

Create a local backup before upgrading:

```bash
python3 scripts/self_host_data.py backup
```

Then pull the release image:

```bash
STUDY_ANYTHING_IMAGE_TAG=v0.2.13-alpha \
USE_PUBLISHED_IMAGES=true \
./scripts/launch_self_host.sh
```

## Known Limitations

- The alpha remains self-host-first. Managed hosting, accounts, billing, Publish, and Teams are not
  included yet.
- There is no shippable standalone Web UI in this release. The supported product path is API, Skill,
  CLI, and platform-agent integration.
- Platform packs are checked-in starter assets, not submitted third-party marketplace listings.
- `agent-eval` exports redacted evidence and adapter strategy. It does not run judge models by itself.
- Promptfoo external eval is available through the wrapper, but Node/npm package installation remains
  an explicit operator action.
- Kimi browser-only chat still cannot operate local scripts or reach `127.0.0.1`; use the documented
  local gateway or a terminal-capable Agent.
- Study Sync is still a local encrypted package boundary, not hosted remote sync.
- Recovery remains manual. API status surfaces do not execute destructive restore operations.
- Plugin installation remains local and explicit; remote marketplace distribution is not implemented.
- First-run GHCR downloads can take several minutes on slower connections.
