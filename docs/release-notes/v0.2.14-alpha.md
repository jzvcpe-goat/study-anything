# v0.2.14-alpha

Study Anything v0.2.14-alpha ships layered teaching Agent orchestration. This release turns the
multi-model teaching idea into a concrete API and platform-tool contract: Study Anything can split one
source into overview, glossary, examples, and Obsidian-style note tasks while the user's own Agent
Gateway decides which model or sub-agent handles each layer.

## Highlights

- Adds Agent capabilities for layered teaching:
  - `teach.overview`
  - `teach.glossary`
  - `teach.examples`
  - `note.scribe`
- Adds `POST /v1/sessions/{session_id}/teaching-layers` for optional pre-quiz teaching output.
- Keeps real model choice outside Study Anything. Different layers can be routed to different
  user-owned HTTP Agent providers without storing model API keys.
- Extends the deterministic fake Agent and mock HTTP Agent so tests and demos cover the new teaching
  task types.
- Adds platform-tool manifest support for `study_anything_teaching_layers` and regenerates the
  OpenAPI, OpenAI-compatible tools, catalog, and platform bundle manifest.
- Updates `scripts/verify_platform_agent_tools.py` so platform wrappers prove the teaching-layer tool
  works over a real HTTP API before the normal quiz, grading, mastery, audit, and eval loop.
- Adds API tests for demo-agent teaching layers and split routing across separate user-owned Agent
  providers.
- Updates default published-image tags to `v0.2.14-alpha`.

## First Launch

Install Docker Desktop or Docker Engine with the Compose plugin, start the daemon, then run:

```bash
python3 scripts/setup_env.py
USE_PUBLISHED_IMAGES=true ./scripts/launch_self_host.sh
```

Open:

- API docs: http://localhost:8000/docs
- API health: http://localhost:8000/v1/health
- System status: http://localhost:8000/v1/system/status

Maintainers can verify the public API image end to end with:

```bash
python3 scripts/verify_published_image_launch.py --tag v0.2.14-alpha
```

## Platform Agent Flow

After attaching a source, a platform Agent can request teaching layers:

```bash
POST /v1/sessions/SESSION_ID/teaching-layers
```

Example request:

```json
{
  "layers": ["overview", "glossary"],
  "language": "zh",
  "level": "beginner"
}
```

Then continue the normal learning loop:

```bash
POST /v1/sessions/SESSION_ID/run
POST /v1/sessions/SESSION_ID/answers
GET /v1/sessions/SESSION_ID/agent-audit
GET /v1/sessions/SESSION_ID/agent-eval/artifact
```

Teaching-layer output is private learning data. Platform wrappers should redact it before logging or
sharing. The Agent audit and eval artifact remain redacted evidence surfaces.

## Upgrade

Create a local backup before upgrading:

```bash
python3 scripts/self_host_data.py backup
```

Then pull the release image:

```bash
STUDY_ANYTHING_IMAGE_TAG=v0.2.14-alpha \
USE_PUBLISHED_IMAGES=true \
./scripts/launch_self_host.sh
```

## Known Limitations

- The alpha remains self-host-first. Managed hosting, accounts, billing, Publish, and Teams are not
  included yet.
- There is no shippable standalone Web UI in this release. The supported product path is API, Skill,
  CLI, and platform-agent integration.
- Teaching-layer outputs are not quality-judged by Study Anything itself. Use the redacted eval
  artifact with Promptfoo, DeepEval, LangChain AgentEvals, or Ragas for external quality evals.
- Kimi browser-only chat still cannot operate local scripts or reach `127.0.0.1`; use the documented
  local gateway or a terminal-capable Agent.
- Study Sync is still a local encrypted package boundary, not hosted remote sync.
- Recovery remains manual. API status surfaces do not execute destructive restore operations.
- Plugin installation remains local and explicit; remote marketplace distribution is not implemented.
- First-run GHCR downloads can take several minutes on slower connections.
