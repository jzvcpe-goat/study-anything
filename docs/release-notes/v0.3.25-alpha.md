# Study Anything v0.3.25-alpha

This alpha adds a release-asset-only bootstrap path for external platform
adoption. The goal is to let a Kimi, Codex, WorkBuddy, or generic HTTP-tool
operator start from the GitHub Release page instead of a development checkout.

## Highlights

- Added `scripts/bootstrap_from_release.py`, which downloads public release
  assets, verifies GitHub digests, extracts the adoption pack, checks platform
  import manifests, and emits a redacted bootstrap transcript.
- Added `release-asset-bootstrap-v1` evidence assets for release review and
  platform submission.
- Added recovery classifications for missing assets, digest mismatch, corrupted
  packs, tool manifest drift, local API unavailability, published-image
  unavailability, and non-ASCII path risk.
- Wired bootstrap evidence into the platform adoption pack, ecosystem
  submission, release gate, and operator docs.

## Validation

```bash
python3 scripts/bootstrap_from_release.py --tag v0.3.25-alpha --runtime metadata-only
python3 scripts/bootstrap_from_release.py --fixture fixtures/release-asset-adoption/asset-only-pass.json --asset-dir platform/generated --runtime metadata-only
python3 scripts/generate_release_asset_bootstrap.py --check
python3 scripts/generate_platform_adoption_pack.py --check
python3 scripts/verify_ecosystem_submission_pack.py
./scripts/release_check.sh
```

## Known Limits

- Metadata-only bootstrap proves release asset integrity and platform import
  readiness. It does not start a runtime unless `--runtime skill-mode` or
  `--runtime published-image` is selected.
