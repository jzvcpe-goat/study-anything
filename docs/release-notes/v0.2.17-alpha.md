# v0.2.17-alpha

Study Anything v0.2.17-alpha moves the project from adoption readiness toward platform-Agent
use. The release focuses on Kimi/Codex/WorkBuddy-style tool surfaces, quality proof for learning
Agent output, Learning Enrichment inputs, and copy-ready Obsidian Markdown export.

## Added

- Adds `POST /v1/sessions/{session_id}/enrichment` so a platform Agent can attach web pages,
  documents, app context, or video slices before the learning loop.
- Adds `GET /v1/sessions/{session_id}/agent-eval/quality`, a redacted deterministic quality report
  that separates invocation proof, contract validity, source binding, teaching layers, quiz quality,
  grading quality, and synthesis quality.
- Adds `GET /v1/evals/quality/cases` for fixed quality cases covering whole-topic teaching,
  glossary/term explanations, quiz generation, grading, synthesis, and source binding.
- Adds `GET /v1/sessions/{session_id}/exports/obsidian` for an Obsidian-compatible Markdown note with
  source references, teaching layers, quiz review, mastery, insights, and enrichment references.
- Adds `evals/deepeval/study_anything_quality_eval.py`, a DeepEval custom-metric adapter with a
  deterministic fallback for environments that have not installed DeepEval yet.

## Changed

- Platform tools now expose enrichment, quality eval, and Obsidian export while continuing to hide
  management endpoints, raw source text, learner answers, feedback, Agent endpoints, and secrets.
- Kimi, Codex, and WorkBuddy packs now document the enrichment-first learning flow, quality gate,
  Obsidian export, and DeepEval runner command.
- `scripts/verify_agent_eval_flow.py` now generates overview and glossary teaching layers before
  quiz/grading/synthesis, then requires `agent-quality-eval-v1` to pass.
- `scripts/run_skill_mode_demo.sh` now runs the quality eval runner as part of the one-command
  local-first smoke test.
- The platform bundle manifest includes the DeepEval adapter and quality eval acceptance command.

## Known Limitations

- The DeepEval adapter can run in native fallback mode when DeepEval is not installed; treat that as a
  compatibility bridge, not as a judge-model benchmark.
- Real model credentials still belong in the user's platform Agent or local gateway, not in Study
  Anything.
- No standalone Web UI is included in the launch path.
- NotebookLM-style knowledge import, LanceDB retrieval evals, hosted Sync/Teams, billing, and
  marketplace flows remain out of scope for this alpha.
