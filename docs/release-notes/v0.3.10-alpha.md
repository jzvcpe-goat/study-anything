# v0.3.10-alpha

Study Anything v0.3.10-alpha adds a first-run lesson authoring kit for
Kimi-compatible, Codex Skill, WorkBuddy-style HTTP, and generic OpenAPI
platform Agents.

## What's New

- Adds `first-run-lesson-authoring-kit-v1`, a redacted, copyable first-lesson
  playbook for external platform Agents.
- Includes Chinese and English kickoff prompts, an operator checklist, a strict
  tool-call sequence, a Learning Context Package template, a user-owned HTTP
  Agent setup snippet, expected output schemas, failure remediation, privacy
  assertions, and evidence export paths.
- Wires the kit into platform packs, ecosystem submission metadata, adoption
  packs, bundle manifests, operator drill evidence, manual-submission rehearsal,
  release checks, and external adoption proof.

## Verification

- `python3 scripts/verify_first_lesson_authoring_kit.py --check`
- `python3 scripts/verify_platform_manual_submission_rehearsal.py --check`
- `python3 scripts/verify_platform_submission_dry_run.py --check`
- `python3 scripts/verify_platform_operator_drill.py --check`
- `python3 scripts/generate_platform_adoption_pack.py --check`
- `python3 scripts/verify_external_adoption.py --pack platform/generated/study-anything-platform-adoption-pack.zip --copy-worktree`
- `./scripts/release_check.sh`

## Known Limits

- The kit is a platform-Agent handoff and verifier, not an official Kimi,
  Codex, WorkBuddy, or marketplace approval.
- Browser-only chat surfaces still cannot call localhost directly.
- Real model credentials and provider routing remain inside the user's own
  Agent or platform runtime.
