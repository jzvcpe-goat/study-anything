# Study Anything v0.3.16-alpha

This alpha hardens external platform adoption after `v0.3.15-alpha` by adding
diagnosable import failures and local-only feedback packages for Kimi, Codex,
WorkBuddy-style HTTP workspaces, and generic OpenAPI tool platforms.

## Highlights

- Added `platform-adoption-feedback-diagnostics-v1`, a release gate for platform
  import, runtime, command, version-drift, endpoint-health, and Agent eval evidence
  diagnostics.
- Added `platform-feedback-package-v1`, a local-only redacted package for sharing
  adoption failure summaries without source text, learner answers, Agent prompts,
  personal profiles, endpoint secrets, model keys, judge keys, or private browser/video
  context.
- Wired the diagnostics and feedback package into release checks, platform packs,
  bundle manifests, ecosystem submission metadata, manual rehearsal, operator drill,
  external adoption proof, and the platform adoption pack.
- Kept real model and judge credentials outside Study Anything. External platform
  Agents and user-owned Agent gateways remain responsible for private credentials,
  browsing, files, application context, video tooling, and external data lookup.

## Verification

Run the normal release gate:

```bash
./scripts/release_check.sh
```

Focused checks:

```bash
python3 scripts/verify_platform_adoption_feedback_diagnostics.py --check
python3 scripts/generate_platform_feedback_package.py --check
python3 scripts/verify_external_adoption.py \
  --pack platform/generated/study-anything-platform-adoption-pack.zip \
  --copy-worktree
```

## Known Limits

- The feedback package is a local handoff artifact, not an automatic telemetry upload.
- Hosted Sync, Publish, Teams, billing, and marketplace payments remain out of scope
  for this release.
- Standalone UI remains intentionally out of scope until the API, Skill, and platform
  Agent adoption path is stable.
