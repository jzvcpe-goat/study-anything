# Study Anything v0.3.28-alpha

This alpha fixes the cleanroom Skill Mode runtime path discovered during the
v0.3.27 release acceptance drill. GitHub source archives may not preserve local
script executable bits, so release-only runtime replay now invokes shell scripts
through `sh` instead of requiring executable file modes.

## Highlights

- Keeps the release-only cleanroom bootloader introduced in v0.3.27.
- Fixes Skill Mode runtime replay from a downloaded GitHub tag source archive.
- Updates clean-clone and external-adoption verifiers so source zip paths do not
  depend on executable bits for `scripts/*.sh`.
- Preserves the platform-first distribution model: six public release assets,
  Kimi/Codex/WorkBuddy/OpenAPI import paths, redacted diagnostics, and no model
  keys stored by Study Anything.

## Validation

```bash
python3 platform/bootstrap/study_anything_release_bootstrap.py --tag v0.3.28-alpha --platform kimi --runtime metadata-only
python3 platform/bootstrap/study_anything_release_bootstrap.py --tag v0.3.28-alpha --platform kimi --runtime skill-mode
python3 scripts/verify_release_asset_adoption.py --tag v0.3.28-alpha --runtime metadata-only
python3 scripts/bootstrap_from_release.py --tag v0.3.28-alpha --runtime metadata-only
python3 scripts/replay_platform_agent_from_release.py --tag v0.3.28-alpha --platform kimi --runtime skill-mode
python3 scripts/generate_platform_adoption_pack.py --check
python3 scripts/verify_ecosystem_submission_pack.py
./scripts/release_check.sh
```

## Known Limits

- Metadata-only mode proves release asset and tool import readiness but does not
  start Study Anything.
- Skill Mode runtime still needs network access the first time it downloads the
  matching GitHub tag source archive and installs Python dependencies.
- Real model choice remains outside Study Anything. The user's platform Agent or
  local Agent gateway owns model credentials, browser access, and external tools.
