# v0.2.23-alpha

Study Anything v0.2.23-alpha is the real external-platform adoption drill
release. It tightens the v0.2.22 adoption pack into a machine-checkable
operator workflow for Kimi Work, Codex, WorkBuddy-style HTTP tool workspaces,
and similar platform Agents.

## Highlights

- Added `scripts/verify_platform_operator_drill.py`, which unpacks the
  platform adoption archive and verifies that Kimi, Codex, and WorkBuddy packs
  reference real import assets, entrypoints, acceptance evidence, and privacy
  constraints.
- Added `platform/generated/study-anything-operator-drill-transcript.json` as
  deterministic evidence that the pack can be consumed as an external platform
  tool directory.
- Wired the operator drill into CI, `scripts/release_check.sh`, and
  `scripts/verify_external_adoption.py`, so `adoption-proof-v1` now includes
  platform-consumption evidence in addition to runtime learning/eval/export
  checks.
- Added `docs/operator-drill.md` for a human operator path that does not require
  a standalone frontend.
- Kept the privacy boundary explicit: real model keys stay in the user's Agent
  or gateway, while Study Anything owns learning state, contracts, audit, eval,
  review memory, and exports.

## Validation Target

Before tagging this release:

```bash
python3 scripts/verify_platform_operator_drill.py --check
python3 scripts/verify_external_adoption.py \
  --pack platform/generated/study-anything-platform-adoption-pack.zip \
  --copy-worktree
./scripts/release_check.sh
python3 scripts/verify_published_image_launch.py --tag v0.2.23-alpha
```

## Known Limits

- This release still treats Kimi/WorkBuddy integrations as platform-pack and
  local HTTP-tool contracts. It does not install inside a proprietary hosted
  Kimi Work or WorkBuddy environment automatically.
- Promptfoo and other external eval runners remain optional/isolated gates with
  explicit timeouts.
- No standalone web UI is required or reintroduced by this release.
