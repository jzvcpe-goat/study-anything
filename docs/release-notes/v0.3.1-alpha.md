# v0.3.1-alpha

Study Anything v0.3.1-alpha turns the platform adoption bundle into a
submission-ready ecosystem pack for Kimi-compatible tools, Codex Skill usage,
WorkBuddy-style HTTP workspaces, and generic OpenAPI tool hosts.

## Added

- `platform/ecosystem-submission.json` with `ecosystem-submission-v1`
  metadata for external platform review.
- `docs/ecosystem-submission.md` explaining the no-frontend, local-first,
  platform-Agent distribution model.
- `scripts/verify_ecosystem_submission_pack.py`, which emits
  `ecosystem-submission-verification-v1` and checks assets, privacy boundaries,
  management endpoint exclusions, generated tool assets, platform packs, and
  the `commercial-readiness-v1` link.

## Changed

- Release defaults now point to `v0.3.1-alpha`.
- The platform bundle and adoption pack include the ecosystem submission
  contract, verifier, and v0.3.1 release notes.
- Release checks now fail if the submission pack drifts from the platform tool
  manifest or exposes high-risk management endpoints.

## Known Limits

- This release still prioritizes local-first platform-Agent usage over a
  standalone Study Anything frontend.
- Hosted sync, hosted publishing, team workspaces, billing, and marketplace
  payments remain future commercial surfaces.
- Kimi browser-only chat still needs a terminal, gateway, or private tool host
  to reach a local Study Anything API.
