# v0.2.25-alpha

Study Anything v0.2.25-alpha is the Learning Enrichment Layer MVP. It keeps the
product platform-agent first: Kimi, Codex, WorkBuddy-style agents, or a local
agent gather browser, document, video, app, Markdown, and Obsidian context;
Study Anything validates that context, preserves source binding, and returns
redacted teaching artifacts.

## Highlights

- Tightens `learning-context-package-v1` and direct enrichment input around
  source type, locator, provenance, redaction policy, excerpt hashes, and
  secret-like value rejection.
- Adds `GET /v1/sessions/{session_id}/exports/enrichment-artifact`, returning a
  `learning-enrichment-artifact-v1` Markdown+HTML micro-lesson for platform
  agents, NotebookLM-style bridges, and Obsidian pipelines.
- Adds `study_anything_enrichment_artifact_export` to the constrained platform
  tool manifest, OpenAPI export, OpenAI-compatible tool definitions, and tool
  catalog.
- Updates bundled importer plugins and the NotebookLM-style fixture so every
  imported context item carries provenance and redaction policy.
- Extends smoke tests and platform verification to check the enrichment artifact
  and raw-text privacy boundary.

## Known Limits

- The enrichment artifact is a deterministic handoff surface, not a standalone
  visual frontend.
- NotebookLM support remains a bridge pattern through user-controlled packages;
  it does not call an official NotebookLM API.
- Platform agents still own browser automation, app access, video slicing, and
  real model credentials.
