# v0.3.29-alpha

This alpha adds the external adopter issue-to-fix support loop on top of the
release-only cleanroom bootstrap from v0.3.28.

## Added

- Added `platform-support-bundle-replay-v1`, a maintainer replay result for
  redacted support bundles, cleanroom reports, and published-image diagnostics.
- Added `scripts/replay_support_bundle.py` to classify failures, block
  privacy-unsafe bundles, and emit copyable GitHub issue bodies.
- Added `platform-support-bundle-replay-evidence-v1` plus fixtures for:
  local GHCR pull timeout, cleanroom runtime launch failure, and privacy
  contract violation.
- Added adoption-pack and release-check coverage so Kimi, Codex, WorkBuddy, and
  generic platform Agents receive the same support bundle replay path.

## Privacy

Support bundles remain manual handoff artifacts. They must not contain raw
source text, learner answers, Agent prompts, Agent endpoint secrets, real model
keys, local absolute paths, personal profiles, or private browser/video/app
context.

## Verification

```bash
python3 scripts/generate_platform_support_bundle_replay.py --check
python3 scripts/verify_platform_support_bundle_replay.py --check
python3 scripts/replay_support_bundle.py \
  --bundle fixtures/platform-support-bundles/local-ghcr-pull-timeout.json \
  --expect-classification local_ghcr_pull_timeout
```
