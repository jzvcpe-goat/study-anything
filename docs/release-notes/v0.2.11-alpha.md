# v0.2.11-alpha

Study Anything v0.2.11-alpha makes the platform-agent path easier to use after the Agent Eval
foundation in v0.2.10-alpha. It adds CLI and Skill commands for redacted Agent proof, plus a dedicated
guide for Codex, Kimi, WorkBuddy-style tools, and terminal-capable Agents.

## Highlights

- Adds `python3 scripts/study_anything_cli.py agent-audit SESSION_ID`.
- Adds `python3 scripts/study_anything_cli.py agent-eval SESSION_ID`.
- Extends the repo-local `study-anything` skill so operating Agents collect Agent proof after learning.
- Extends Skill Mode smoke to verify `agent-audit.status=verified` and
  `agent-eval-artifact-v1`.
- Adds `docs/platform-agent-integrations.md` for Codex, Kimi, WorkBuddy-style tools, and private HTTP
  tool wrappers.
- Documents the acceptance gate for platform integrations: complete a learning loop, return Agent
  audit proof, and return the redacted eval artifact without leaking private learning content.
- Updates default published-image tags to `v0.2.11-alpha`.

## First Launch

Install Docker Desktop or Docker Engine with the Compose plugin, start the daemon, then run:

```bash
python3 scripts/setup_env.py
USE_PUBLISHED_IMAGES=true ./scripts/launch_self_host.sh
```

Open:

- API docs: http://localhost:8000/docs
- API health: http://localhost:8000/v1/health
- System status: http://localhost:8000/v1/system/status

Maintainers can verify the public API image end to end with:

```bash
python3 scripts/verify_published_image_launch.py --tag v0.2.11-alpha
```

## Platform Agent Proof

After a completed learning session:

```bash
python3 scripts/study_anything_cli.py agent-audit SESSION_ID
python3 scripts/study_anything_cli.py agent-eval SESSION_ID
```

`agent-audit` proves required Study Anything learning tasks were handled by an Agent provider.
`agent-eval` emits the redacted bridge for Promptfoo, DeepEval, LangChain AgentEvals, and Ragas.

## Upgrade

Create a local backup before upgrading:

```bash
python3 scripts/self_host_data.py backup
```

Then pull the release image:

```bash
STUDY_ANYTHING_IMAGE_TAG=v0.2.11-alpha \
USE_PUBLISHED_IMAGES=true \
./scripts/launch_self_host.sh
```

## Known Limitations

- The alpha remains self-host-first. Managed hosting, accounts, billing, Publish, and Teams are not
  included yet.
- There is no shippable standalone Web UI in this release. The supported product path is API, Skill,
  CLI, and platform-agent integration.
- `agent-eval` exports a redacted eval artifact; it does not run judge models by itself.
- Kimi browser-only chat still cannot operate local scripts or reach `127.0.0.1`; use the documented
  local gateway or a terminal-capable Agent.
- Study Sync is still a local encrypted package boundary, not hosted remote sync.
- Recovery remains manual. API status surfaces do not execute destructive restore operations.
- Plugin installation remains local and explicit; remote marketplace distribution is not implemented.
- First-run GHCR downloads can take several minutes on slower connections.
