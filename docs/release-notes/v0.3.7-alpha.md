# v0.3.7-alpha

Study Anything v0.3.7-alpha adds an external platform submission dry-run layer.
The goal is to make Kimi-compatible, Codex Skill, WorkBuddy-style HTTP, and
generic OpenAPI handoffs reviewable before manual platform submission.

## What Changed

- Added `platform-submission-dry-run-v1` in
  `scripts/verify_platform_submission_dry_run.py`.
- Added a generated readiness report at
  `platform/generated/study-anything-platform-submission-dry-run.json`.
- Simulates submission readiness for Kimi-compatible OpenAI tools, Codex Skill,
  WorkBuddy-style HTTP tools, and generic OpenAPI imports.
- Reports per-platform status, missing assets, missing acceptance commands,
  warnings, manual submission checklist, and privacy assertions.
- Wired the dry-run report into release checks, external adoption proof,
  ecosystem submission verification, platform packs, operator drill evidence,
  and the distributable adoption pack.

## Verify

```bash
python3 scripts/verify_platform_submission_dry_run.py --check
python3 scripts/verify_ecosystem_submission_pack.py
python3 scripts/verify_platform_ecosystem_packs.py
python3 scripts/generate_platform_bundle_manifest.py --check
python3 scripts/generate_platform_adoption_pack.py --check
./scripts/release_check.sh
```

Expected submission proof:

- all four target platform profiles are present
- no platform is blocked
- required import assets exist
- acceptance commands include the dry-run verifier
- generated report is redacted
- no standalone frontend, billing, hosted service, or model-key custody is
  required for the current launch path

## Known Limits

- This is a local/offline dry-run, not an official approval from Kimi, Codex,
  WorkBuddy, or any marketplace.
- Kimi-compatible and generic OpenAPI hosts may still require manual import
  tweaks depending on their tool schema rules.
- Hosted sync, team workspaces, marketplace payments, and managed submissions
  remain future commercial work.
