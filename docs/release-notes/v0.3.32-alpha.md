# Delivery Clearance Personal Local Alpha v0.3.32-alpha

**未经放行，不得交付。**

Delivery Clearance does not prove that AI is always correct. It proves why this delivery may move
forward, to whom, for what purpose, within what limits, and under whose responsibility.

AI 交付放行协议不证明 AI 永远正确；它证明这次交付为什么可以继续向前、可以交给谁、
可以用于什么、受到哪些限制，以及由谁承担责任。

## Release Scope

This prerelease is the first public Delivery Clearance release profile focused on one operator
auditing one local Git project. Its maximum clearance scope is `personal_local`.

It includes:

- `delivery-clearance init/audit/verify` for state-bound local project receipts;
- `delivery-clearance-plugin-evidence` for deterministic plugin evidence classification;
- explicit responsibility acceptance, short receipt validity, tamper replay, and Git-state binding;
- hard blocking for external-write capability, credential use, observed external mutation,
  undeclared network use, failed required checks, and unsupported scope expansion;
- a real installed-plugin boundary study and machine-verifiable release receipt.

## Installation

The release workflow attaches `study_anything-0.3.32a0-py3-none-any.whl`. The distribution name is
retained for compatibility; the product name is Delivery Clearance.

```bash
python3.11 -m venv .venv
.venv/bin/python -m pip install study_anything-0.3.32a0-py3-none-any.whl
.venv/bin/delivery-clearance --help
.venv/bin/delivery-clearance-plugin-evidence --help
```

## Claim Boundary

This release is local-first, deterministic, and self-attested. It does not prove AI or plugin
correctness, independent review, customer delivery authority, production readiness, legal or
security certification, professional-domain correctness, OS-level sandboxing, future external
actions, or absence of side effects outside the observed evidence boundary.

Configured project checks run with the current user's permissions. Plugin Evidence Adapter does not
execute or install plugins. The release does not authorize external writes or irreversible effects.

## Known Limitations

- The repository slug, Python distribution, Python namespace, and `cbb.*` contracts retain
  historical compatibility identifiers.
- The Personal Local Alpha does not claim a new published Docker image line.
- No independent security audit or real external-adopter attestation has completed.
- Professional judgment and native UI behavior require additional domain or native evidence.
- A complete clean-clone release receipt is mandatory before the tag is created.
