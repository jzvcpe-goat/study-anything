# v0.2.3-alpha

Study Anything v0.2.3-alpha is the recommended self-host release for new local-first trials.

## Highlights

- Adds guided first-run Web onboarding for the deterministic demo loop, learner-owned source material,
  and Bring Your Own Agent setup.
- Adds a bilingual Web Launch panel with local-only PMF signals for completed learning loops, completion
  rate, active learner hashes, repeat usage, mastery delta, plugin readiness, and future-service interest.
- Adds local PMF APIs:
  - `GET /v1/metrics/pmf`
  - `GET /v1/pmf/summary`
  - `POST /v1/pmf/interest`
- Keeps PMF data privacy-first: no reading prose, quiz prompts, answers, grading feedback, insights,
  scribe logs, Agent metadata, raw user IDs, or raw contact values are exposed by the PMF APIs.
- Adds permission-gated Web plugin installation for explicitly selected local plugin directories.
- Expands the self-host doctor with Docker, Compose config, profile port, health, Agent gateway,
  plugin-directory, and recovery-command checks.
- Adds Skill Mode foreground launch for desktop or agent environments that do not preserve background
  processes:

  ```bash
  ./scripts/launch_skill_mode.sh --foreground
  ```

- Retains the Bring Your Own Agent learning loop, Postgres state, optional FalkorDB projection,
  backup/restore tooling, and multi-architecture GHCR images.

## First Launch

Install Docker Desktop or Docker Engine with the Compose plugin, start the daemon, then run:

```bash
python3 scripts/setup_env.py
USE_PUBLISHED_IMAGES=true ./scripts/launch_self_host.sh
```

Open http://localhost:5173.

After launch, verify the public API flow:

```bash
API_BASE=http://127.0.0.1:8000 python3 scripts/verify_full_api_flow.py
```

## Upgrade

Create a local backup before upgrading:

```bash
python3 scripts/self_host_data.py backup
```

Then pull the release images:

```bash
STUDY_ANYTHING_IMAGE_TAG=v0.2.3-alpha \
USE_PUBLISHED_IMAGES=true \
./scripts/launch_self_host.sh
```

## Known Limitations

- The alpha remains self-host-first. Managed hosting, accounts, billing, Sync, Publish, and Teams are
  not included yet.
- PMF metrics are local aggregate signals, not hosted product analytics.
- First-run GHCR downloads can take several minutes on slower connections.
- Agent provider defaults remain stored in the local `study_anything_data` Docker volume.
- Plugin installation is local and explicit. Remote marketplace review, signing, and update UX are not
  implemented yet.
- LanceDB retrieval and Playwright browser automation remain planned for the next alpha.
