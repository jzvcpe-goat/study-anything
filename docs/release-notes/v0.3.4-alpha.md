# v0.3.4-alpha

Study Anything v0.3.4-alpha hardens the NotebookLM, Obsidian, and Learning
Enrichment bridge. The goal is to let platform Agents collect web pages,
documents, video slices, app context, Markdown, and Obsidian notes, then hand
Study Anything a bounded learning context package without turning the project
into a raw-data vault or model-key custodian.

## Added

- `notebooklm-obsidian-bridge-hardening-v1` via
  `scripts/verify_notebooklm_obsidian_bridge_hardening.py`.
- Hidden instruction detection for Learning Context Package text, metadata,
  provenance, and nested values.
- Idempotent duplicate context item handling with conflict rejection for reused
  `item_id` values that point at different content.
- Release, adoption, ecosystem submission, and platform pack gates for the new
  bridge hardening verifier.

## Changed

- `learning-package-v1` now exports only a public Agent summary for teaching
  layers and excludes raw Agent metadata or endpoints.
- Learning context package validation recursively rejects secret-like keys,
  secret-like values, and hidden/system prompt-like text in nested metadata.
- Platform packs now require bridge-hardening evidence alongside gateway
  hardening, eval, retrieval, Obsidian, learning package, and second-brain
  handoff evidence.

## Verification

```bash
python3 scripts/verify_notebooklm_obsidian_bridge_hardening.py
python3 scripts/verify_agent_gateway_hardening.py
python3 scripts/verify_ecosystem_submission_pack.py
python3 scripts/verify_platform_ecosystem_packs.py
./scripts/release_check.sh
```

## Known Limits

- NotebookLM remains a manual bridge; no official NotebookLM API is required or
  assumed.
- Direct Obsidian and learning package exports remain user-owned exports and may
  include learner answers when the user asks for that. The stricter
  `second-brain-handoff-v1` excludes learner answers, grading feedback, raw
  source text, raw enrichment text, Agent metadata, endpoints, and secrets.
- This release does not add a standalone graph or NotebookLM UI.
