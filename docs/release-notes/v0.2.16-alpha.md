# v0.2.16-alpha

Study Anything v0.2.16-alpha is an external-user adoption readiness release. It focuses on proving
that a user can clone the repository, run the local-first Skill/API path, connect platform agents, and
collect redacted Agent evidence before adding real model credentials.

## Added

- Adds `scripts/verify_clean_clone_adoption.py`, a disposable checkout verifier that:
  - clones or copies the repo into a temporary directory
  - generates `.env`
  - starts Skill Mode
  - completes the Skill CLI flow
  - verifies teaching layers, quiz, grading, mastery, `agent-audit`, and `agent-eval/artifact`
  - verifies the OpenAI-compatible gateway dry-run flow
  - can optionally invoke Promptfoo through `--with-promptfoo`
- Adds `scripts/diagnose_adoption.py` for common external adoption blockers:
  - localhost API unreachable
  - Docker daemon unavailable
  - GHCR manifest unavailable or slow
  - HTTP Agent endpoint unhealthy
  - provider capability defaults missing
- Adds a published-image smoke fallback:
  - `scripts/verify_published_image_launch.py --pull-timeout-seconds ... --allow-pull-timeout-report`
  - returns an explicit `blocked_by_local_ghcr_pull` diagnostic when local GHCR layer downloads are the
    bottleneck.
- Adds `docs/adoption.md` as the operator guide for clean-clone smoke, Promptfoo, platform packs,
  diagnostics, and GHCR fallback evidence.

## Changed

- Platform packs for Codex, Kimi-compatible agents, and WorkBuddy-style HTTP workspaces now include
  clean-clone adoption and diagnostics commands in their machine-readable metadata.
- Platform generated assets and the platform bundle manifest include the adoption guide, verifier,
  diagnostics helper, Promptfoo config, Skill Mode scripts, CLI, and external eval wrapper.
- CI runs the real clean-clone adoption smoke.
- `release_check.sh` runs the current worktree adoption smoke plus diagnostics before release.
- Skill Mode demo now passes the active API base to platform-agent verification, so random-port adoption
  checks do not accidentally call `127.0.0.1:8000`.
- Agent eval docs now distinguish invocation proof, redacted artifact readiness, Promptfoo contract
  gates, and later quality evaluation with tools such as DeepEval, LangChain AgentEvals, and Ragas.

## Verification

Release candidates should pass:

```bash
./scripts/release_check.sh
python3 scripts/verify_clean_clone_adoption.py --repo .
python3 scripts/verify_clean_clone_adoption.py --repo . --with-promptfoo --promptfoo-required
python3 scripts/verify_platform_ecosystem_packs.py
python3 scripts/generate_platform_bundle_manifest.py --check
python3 scripts/verify_agent_eval_assets.py
```

After GHCR publish:

```bash
docker manifest inspect ghcr.io/jzvcpe-goat/study-anything/api:v0.2.16-alpha
python3 scripts/verify_published_image_launch.py --tag v0.2.16-alpha
```

If the local image pull is too slow, pair successful GitHub `docker-images` workflow evidence and
manifest inspection with:

```bash
python3 scripts/verify_published_image_launch.py \
  --tag v0.2.16-alpha \
  --pull-timeout-seconds 180 \
  --allow-pull-timeout-report
```

## Known Limitations

- No standalone Web UI is included in the launch path.
- Promptfoo checks the redacted artifact contract and native gates; it is not a full teaching-quality
  judge.
- Real model credentials stay outside Study Anything in the user's Agent gateway or platform Agent.
- Published-image smoke still depends on local Docker and network access unless verified through
  GitHub Actions or another remote runner.
- Hosted Sync, Publish, Teams, billing, and marketplace flows are intentionally out of scope for this
  alpha.
