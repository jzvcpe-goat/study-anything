# v0.2.26-alpha

Study Anything v0.2.26-alpha hardens the second-brain handoff layer. It keeps
the product local-first and platform-Agent-first: Kimi, Codex, WorkBuddy, or a
user-owned Agent still owns browsing, files, video slicing, tools, and model
credentials; Study Anything owns learning state, validation, audit, eval, and
redacted reusable learning artifacts.

## Added

- `GET /v1/sessions/{session_id}/exports/second-brain-handoff` returns
  `second-brain-handoff-v1`.
- `second-brain-obsidian-note-v1` contains frontmatter, backlinks, note graph
  references, source map, learning map, mastery snapshot, and review queue
  metadata.
- `second-brain-archive-manifest-v1` records deterministic file hashes for
  Obsidian Markdown, redacted learning package JSON, and enrichment Markdown/HTML
  files.
- CLI command:

```bash
python3 scripts/study_anything_cli.py second-brain-handoff SESSION_ID --archive-dir /tmp/study-anything-archive
```

- Platform tool:
  `study_anything_second_brain_handoff_export`.

## Privacy Boundary

The second-brain handoff excludes raw source text, raw enrichment text, learner
answers, grading feedback, Agent endpoints, raw Agent metadata, and secrets.
Use it as the default Kimi/Codex/WorkBuddy-visible long-term memory export.

The older Obsidian and learning-package exports remain available for user-owned
workflows, but platform wrappers should treat them as private learning data.

## Known Limits

- NotebookLM support is a manual bridge contract. Study Anything does not depend
  on an official NotebookLM API.
- Hosted Sync, Teams, Publish, marketplace payments, and enterprise SSO remain
  post-PMF commercial-service work.
- The standalone frontend is not part of this release path.
