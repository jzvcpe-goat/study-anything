# v0.3.0-alpha

Study Anything v0.3.0-alpha makes commercial readiness machine-readable while keeping the product
OSS, self-host-first, API/Skill Mode first, and platform-Agent friendly.

## Added

- Added `commercial-readiness-v1`, a metadata-only contract for commercial launch status,
  local-first invariants, hosted-service boundaries, PMF prerequisites, monetization alignment, and
  privacy limits.
- Added `GET /v1/commercial/readiness` for Kimi, Codex, WorkBuddy-style tools, and local Agents to
  answer what can launch now:
  - GitHub OSS launch: ready.
  - Self-host Alpha: ready.
  - Platform Agent distribution: ready.
  - Standalone app: not in the launch path.
  - Hosted paid services: not ready.
- Added `study_anything commercial-readiness` to the CLI.
- Added `scripts/verify_commercial_readiness.py`, now included in `scripts/release_check.sh`.
- Added `study_anything_commercial_readiness` to the platform tool manifest, generated OpenAPI,
  OpenAI-compatible tool definitions, tool catalog, platform bundle, and adoption pack.

## Commercial Boundaries

- Hosted Sync, Publish, Teams, and Catalyst are now explicit `contract_only` services. Each service
  lists the local foundation, what must not block the OSS core, and what must exist before sale.
- The free local core remains self-hosted learning, Skill Mode, HTTP API, fake deterministic Agent,
  user-owned HTTP Agent integration, community plugins, and local exports.
- Paid services should sell convenience, reliability, collaboration, hosting, and trust operations,
  not lock-in to the learning workflow or custody of real model credentials.

## Verification

Recommended local gates:

```bash
.venv/bin/python -m unittest discover apps/api/tests
STUDY_ANYTHING_STRICT_MSGPACK=1 .venv/bin/python -m unittest discover apps/api/tests
.venv/bin/python scripts/verify_commercial_readiness.py
./scripts/release_check.sh
```

Platform operators can verify the runtime contract with:

```bash
API_BASE=http://127.0.0.1:8000 python3 scripts/verify_platform_agent_tools.py
```

## Known Limits

- This release does not implement billing, hosted accounts, SSO, hosted Sync, hosted Publish, hosted
  Teams, plugin marketplace payments, or a standalone frontend.
- `commercial-readiness-v1` is a launch-boundary and architecture gate. It is not PMF proof by
  itself.
- PMF remains local and opt-in until maintainers have real adoption evidence from external users.
