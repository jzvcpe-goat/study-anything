# v0.2.7-alpha

Study Anything v0.2.7-alpha is a corrective self-host release focused on making the public image path
trustworthy before broader MVP trials.

## Highlights

- Fixes API runtime version reporting so `/v1/health`, `/v1/system/status`, and the package metadata
  stay aligned.
- Updates published-image defaults to the corrected `v0.2.7-alpha` images.
- Adds `scripts/verify_published_image_launch.py`, a disposable GHCR smoke that:
  - creates an isolated Compose project with random ports
  - pulls the published API and Web images
  - checks the running API version
  - completes the public API learning flow
  - completes the Web same-origin learning, Sync, recovery, PMF, and plugin-trust smoke
  - removes temporary containers and volumes on success
- Refines the Learn surface toward a natural-language-first study console while keeping source editing,
  progress, Agent status, HITL, and demo flow backed by the real API.

## First Launch

Install Docker Desktop or Docker Engine with the Compose plugin, start the daemon, then run:

```bash
python3 scripts/setup_env.py
USE_PUBLISHED_IMAGES=true ./scripts/launch_self_host.sh
```

Open http://localhost:5173.

Maintainers can verify the public images end to end with:

```bash
python3 scripts/verify_published_image_launch.py --tag v0.2.7-alpha
```

## Upgrade

Create a local backup before upgrading:

```bash
python3 scripts/self_host_data.py backup
```

Then pull the release images:

```bash
STUDY_ANYTHING_IMAGE_TAG=v0.2.7-alpha \
USE_PUBLISHED_IMAGES=true \
./scripts/launch_self_host.sh
```

## Known Limitations

- The alpha remains self-host-first. Managed hosting, accounts, billing, Publish, and Teams are not
  included yet.
- Study Sync is still a local encrypted package boundary, not hosted remote sync.
- Recovery remains manual. Web/API status surfaces do not execute destructive restore operations.
- Plugin installation remains local and explicit; remote marketplace distribution is not implemented.
- First-run GHCR downloads can take several minutes on slower connections.
- LanceDB retrieval and Playwright browser automation remain planned for a later alpha.
