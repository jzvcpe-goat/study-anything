# v0.3.3-alpha

Study Anything v0.3.3-alpha hardens the user-owned Agent gateway path so Kimi,
Codex, WorkBuddy-style workspaces, and private platform Agents can use Study
Anything without handing model keys to the project.

## Added

- `agent-gateway-hardening-verification-v1` via
  `scripts/verify_agent_gateway_hardening.py`.
- Gateway `/health` now declares capabilities and a privacy contract.
- Gateway `/invoke` rejects unsupported task types and malformed task shapes
  before reaching the upstream model.
- Agent health responses now include `diagnostic_code`, `latency_ms`, and
  explicit privacy flags.
- Release checks and external adoption proof now run the gateway hardening
  verifier.

## Changed

- Agent provider registration rejects endpoint credentials, secret-like query
  parameters, and secret metadata keys. Real credentials must stay inside the
  user's gateway or platform Agent environment.
- Agent provider status returns URL-level redaction for endpoints and redacted
  metadata only.
- Legacy local provider config with unsafe endpoint or metadata secrets is
  sanitized, disabled, and written back without the secret value.
- Ecosystem submission and adoption packs now include the gateway hardening
  verifier.

## Verification

```bash
python3 scripts/verify_agent_gateway_hardening.py
python3 scripts/verify_openai_compatible_gateway.py --gateway-only
python3 scripts/verify_ecosystem_submission_pack.py
./scripts/release_check.sh
```

## Known Limits

- The CLI Agent adapter remains disabled by default.
- MCP Agent adapters remain plugin ecosystem extension points.
- Real teaching quality still depends on the user's own Agent and model stack.
