# v0.2.4-alpha

Study Anything v0.2.4-alpha is the recommended self-host release for new local-first trials.

## Highlights

- Adds explicit-consent PMF export packages for community feedback and hosted waitlist review:
  - `POST /v1/pmf/export`
  - returns `409` unless `consent_to_share=true`
  - returns `schema_version=pmf-export-v1` after consent
- Keeps PMF exports aggregate-only. The package excludes source text, quiz prompts, answers, grading
  feedback, insights, scribe logs, Agent metadata, raw user identifiers, raw contact values, individual
  contact hashes, individual user hashes, and freeform comments.
- Adds a bilingual Web Launch panel consent gate for generating a shareable PMF package.
- Extends the full API smoke script to verify PMF export consent failure and success paths.
- Adds actionable Docker diagnostics for source builds from non-ASCII checkout paths. Docker Desktop
  BuildKit/buildx can fail before the app build starts with:

  ```text
  header key "x-docker-expose-session-sharedkey" contains value with non-printable ASCII characters
  ```

  The self-host launcher and doctor now explain the recovery path instead of surfacing only the Docker
  gRPC error.
- Retains the Bring Your Own Agent learning loop, Postgres state, optional FalkorDB projection,
  backup/restore tooling, first-run Web onboarding, Skill Mode, and multi-architecture GHCR images.

## First Launch

Install Docker Desktop or Docker Engine with the Compose plugin, start the daemon, then run:

```bash
python3 scripts/setup_env.py
USE_PUBLISHED_IMAGES=true ./scripts/launch_self_host.sh
```

Open http://localhost:5173.

If you want to build from source instead of using published images, keep the checkout path ASCII-only:

```bash
git clone https://github.com/jzvcpe-goat/study-anything.git ~/study-anything
cd ~/study-anything
python3 scripts/setup_env.py
./scripts/launch_self_host.sh
```

After launch, verify the public API flow:

```bash
API_BASE=http://127.0.0.1:8000 python3 scripts/verify_full_api_flow.py
```

## Upgrade

Create a local backup before upgrading:

```bash
python3 scripts/self_host_data.py backup
```

Then pull the release images:

```bash
STUDY_ANYTHING_IMAGE_TAG=v0.2.4-alpha \
USE_PUBLISHED_IMAGES=true \
./scripts/launch_self_host.sh
```

## PMF Export

The PMF export is local and user-controlled. It is meant for users who explicitly choose to share
aggregate self-host usage signals with maintainers or the community.

```bash
curl -X POST http://localhost:8000/v1/pmf/export \
  -H 'Content-Type: application/json' \
  -d '{"consent_to_share":true,"destination":"self_archive"}'
```

## Known Limitations

- The alpha remains self-host-first. Managed hosting, accounts, billing, Sync, Publish, and Teams are
  not included yet.
- PMF exports are local aggregate packages, not hosted product analytics.
- First-run GHCR downloads can take several minutes on slower connections.
- Agent provider defaults remain stored in the local `study_anything_data` Docker volume.
- Plugin installation is local and explicit. Remote marketplace review, signing, and update UX are not
  implemented yet.
- The Web Launch panel is functional but still visually dense in desktop layouts.
- LanceDB retrieval and Playwright browser automation remain planned for the next alpha.
