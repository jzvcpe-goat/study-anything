# Study Anything v0.3.30-alpha

Study Anything v0.3.30-alpha hardens multi-teacher Agent eval evidence. It makes the learning loop
prove that layered teaching actually used the Study Anything Agent contract before platform Agents
or external eval tools trust the result.

## Added

- `scripts/verify_multiteacher_agent_eval_hardening.py` emits
  `multiteacher-agent-eval-hardening-v1`.
- The verifier covers both the deterministic fake Agent path and a mock user-owned HTTP Agent path.
- Multi-teacher attribution now has an explicit release gate for:
  - `teach.overview`
  - `teach.glossary`
  - `quiz.generate`
  - `answer.grade`
  - `insight.synthesize`
- Each required layer must expose redacted `provider_id`, `task_type`, `status`, `latency_ms`, and
  optional `confidence` evidence.
- Negative gates cover missing teaching attribution, forged provider ids, missing external judge
  runtime in optional versus required mode, and privacy redaction failures.

## Changed

- `agent-audit-v1`, `agent-eval-artifact-v1`, and `agent-eval-policy-v1` now treat the five
  multi-teacher tasks as the native required task set.
- `scripts/release_check.sh` now runs the multi-teacher Agent eval hardening verifier.
- `docs/agent-eval.md`, `docs/platform-agent-integrations.md`, `docs/use-with-kimi.md`,
  `docs/release-checklist.md`, and `docs/commercial-readiness.md` document the whole-topic,
  glossary, active-recall, feedback, and synthesis layer contract.

## Privacy

- The verifier must not return raw source text, learner answers, Agent endpoints, raw Agent
  metadata, real model keys, judge keys, or secret-like metadata values.
- External judge and model credentials remain outside Study Anything, inside the user's Agent, CI,
  platform Agent, or external eval workspace.

## Verify

```bash
python3 scripts/verify_multiteacher_agent_eval_hardening.py
./scripts/release_check.sh
```

## Known Limits

- This release proves attribution and minimum native eval readiness. It does not add a standalone UI.
- Multi-model routing still belongs inside the user-owned Agent gateway or platform Agent. Study
  Anything records task evidence and validates outputs; it does not choose or store real model
  credentials.
