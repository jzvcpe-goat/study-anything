# v0.2.2-alpha

Study Anything v0.2.2-alpha is the recommended self-host release for new local trials.

## Highlights

- Shows Docker layer progress during published-image pulls so slow first-run downloads remain
  understandable.
- Keeps sequential API and Web pulls to make cold downloads easier to inspect.
- Retains the one-command GHCR-backed core launch:

  ```bash
  USE_PUBLISHED_IMAGES=true ./scripts/launch_self_host.sh
  ```

- Keeps source builds as the default local-first path.
- Supports explicit image tags, registry mirrors, and offline cached-image launches.
- Retains the Bring Your Own Agent learning loop, Postgres state, optional FalkorDB projection,
  backup/restore tooling, and `linux/amd64` plus `linux/arm64` images from v0.2.1-alpha.

## First Launch

Install Docker Desktop or Docker Engine with the Compose plugin, start the daemon, then run:

```bash
python3 scripts/setup_env.py
USE_PUBLISHED_IMAGES=true ./scripts/launch_self_host.sh
```

Open http://localhost:5173.

## Upgrade

Create a local backup before upgrading:

```bash
python3 scripts/self_host_data.py backup
```

Then pull the patch release images:

```bash
STUDY_ANYTHING_IMAGE_TAG=v0.2.2-alpha \
USE_PUBLISHED_IMAGES=true \
./scripts/launch_self_host.sh
```

## Known Limitations

- The alpha remains self-host-first. Managed hosting, accounts, billing, Sync, Publish, and Teams are
  not included yet.
- First-run GHCR downloads can take several minutes on slower connections.
- Agent provider defaults remain stored in the local `study_anything_data` Docker volume.
- Plugin installation is local and explicit. Remote marketplace review, signing, and install UI are
  not implemented yet.
- LanceDB retrieval and Playwright browser automation remain planned for the next alpha.
