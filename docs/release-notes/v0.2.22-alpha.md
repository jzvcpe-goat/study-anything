# v0.2.22-alpha

Study Anything v0.2.22-alpha is the external-adoption proof release. It packages the
local-first learning layer so Kimi Work, Codex, WorkBuddy-style HTTP workspaces, and other
platform Agents can adopt it without relying on the deleted standalone frontend.

## Highlights

- Added the distributable `study-anything-platform-adoption-pack.zip` with a SHA256 manifest,
  OpenAPI/OpenAI tool import assets, platform packs, Skill files, gateway examples, fixtures,
  verifier scripts, and operator documentation.
- Added `scripts/verify_external_adoption.py`, which emits `adoption-proof-v1` after starting
  Skill Mode from a current worktree, copied worktree, or clean clone and exercising the external
  platform Agent flow.
- Covered importer, enrichment, retrieval search, retrieval eval, teaching layers, agent eval,
  Obsidian export, and NotebookLM-style learning-package export in the adoption verifier.
- Documented Kimi Work, Codex, and WorkBuddy-style integration as the preferred product surface:
  the platform Agent owns browsing, files, video slicing, outside tools, real model credentials,
  and conversation while Study Anything owns source-bound learning workflow and audit evidence.
- Strengthened adoption diagnostics for GHCR/Docker Hub availability, non-ASCII paths, port
  conflicts, local Agent endpoints, Node/npm, and optional Promptfoo failures.

## Privacy Boundary

- Study Anything still stores no real model API keys. Keep provider keys and tool credentials in
  the user's own Agent or platform runtime.
- Adoption proof is redacted. It must not include raw long source text, private browsing/video
  context, learner answer bodies, Agent endpoint secrets, or platform credentials.
- NotebookLM and Obsidian handoff remain user-owned exports. They are bridges for learning memory,
  not hosted lock-in.

## Known Limitations

- This release proves platform adoption and local runtime evidence; it does not restore a standalone
  frontend.
- Promptfoo remains an optional external quality gate unless explicitly required by the operator.
- Production remote hosting, billing, team workspaces, and marketplace payment flows remain future
  commercial services after adoption and PMF evidence.
