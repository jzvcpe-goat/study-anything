# v0.3.5-alpha

Study Anything v0.3.5-alpha hardens the plugin ecosystem path with
quarantine-first local installation. The goal is public OSS distribution where
community plugins are reviewable by default instead of silently becoming active.

## What Changed

- Added `plugin-quarantine-verification-v1` in `scripts/verify_plugin_quarantine.py`.
- Changed plugin install API behavior so `POST /v1/plugins/install` quarantines by default after
  exact permission confirmation.
- Added explicit approved install flow with `approve_install=true`.
- Added CLI default quarantine behavior in `scripts/install_local_plugin.py`; use
  `--approve-install` for the final install copy.
- Expanded `plugin-trust-v1` with lifecycle states, quarantine policy, and manual approval
  requirements.
- Blocked `do_not_install` trust recommendations before both quarantine and install copies.
- Added tests for API quarantine, approved install, blocked digest mismatch, SDK validation,
  trust policy, and registry quarantine behavior.
- Wired the quarantine verifier into release checks, external adoption proof, ecosystem submission,
  platform packs, operator drill evidence, and the adoption pack.

## Verify

```bash
python3 scripts/verify_plugin_quarantine.py
python3 scripts/verify_ecosystem_submission_pack.py
python3 scripts/verify_platform_ecosystem_packs.py
python3 scripts/generate_platform_bundle_manifest.py --check
python3 scripts/generate_platform_adoption_pack.py --check
./scripts/release_check.sh
```

Expected quarantine proof:

- unknown local plugins return `lifecycle_status=quarantined` by default
- quarantined plugins do not appear in `GET /v1/plugins`
- explicit `approve_install=true` returns `lifecycle_status=installed`
- registry digest mismatches return `do_not_install` and `409`
- preview and quarantine do not execute plugin entrypoints

## Known Limits

- This is still a local-first alpha, not a hosted marketplace.
- Registry signatures are supported as local trust metadata, but there is no hosted maintainer review
  queue yet.
- Quarantine is a review buffer; it does not sandbox arbitrary plugin code after a user approves and
  later runs a plugin runtime.
- External platform tool packs intentionally do not expose plugin installation endpoints.
