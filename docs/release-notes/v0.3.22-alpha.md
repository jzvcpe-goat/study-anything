# Study Anything v0.3.22-alpha

This release adds a dedicated published-image evidence layer for external
operators. The goal is to stop treating every local GHCR pull timeout as an
ambiguous deployment failure.

## Added

- `published-image-evidence-v1` generated JSON, Markdown, zip, and checksum.
- `published-image-evidence-fixture-v1` fixtures for local pull timeout,
  missing manifest platform, docker-images failure, GHCR unavailable, remote
  smoke pass, and remote smoke failure.
- `scripts/generate_published_image_evidence.py` and
  `scripts/verify_published_image_evidence.py`.
- Adoption-pack, ecosystem-submission, operator-drill, manual-rehearsal, and
  external-adoption proof wiring for the new evidence layer.

## Acceptance

```bash
python3 scripts/generate_published_image_evidence.py --check
python3 scripts/verify_published_image_evidence.py --check
python3 scripts/verify_published_image_evidence.py \
  --pack platform/generated/study-anything-platform-adoption-pack.zip
```

The evidence is metadata-only. It does not include learning content, Agent
endpoints, local absolute paths, model keys, or private support payloads.
