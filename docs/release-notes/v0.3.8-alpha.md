# v0.3.8-alpha

Study Anything v0.3.8-alpha hardens the real external Agent evaluation path.
The release keeps the product local-first and platform-agent-first: Study
Anything does not store real model keys or own the user's browsing/tool
surface, but it can now prove that a user-owned HTTP Agent produced safe,
auditable learning evidence.

## Added

- `scripts/verify_external_agent_adapter_hardening.py` emits
  `external-agent-adapter-hardening-v1`.
- The verifier separates fake deterministic evidence from external Agent
  evidence and checks `used_external_agent=true` with `used_fake_agent=false`.
- The verifier covers bad external Agent outputs: malformed JSON, invalid
  status, missing content, invalid score, invalid confidence, timeout, missing
  citations, and missing declared capability.
- Secret-looking string values in Agent metadata are redacted even when they
  appear under non-secret field names.

## Changed

- `release_check.sh`, external adoption proof, platform packs, ecosystem
  submission metadata, bundle manifests, and adoption packs now include the new
  external Agent adapter hardening gate.
- Platform pack evidence now declares
  `external_agent_adapter_hardening.schema_version == external-agent-adapter-hardening-v1`.

## Privacy

- The hardening report must not return raw source text, learner answers, Agent
  endpoints, endpoint secrets, model keys, or secret-looking metadata values.
- Real model credentials remain inside the user's Agent or platform environment.

## Known Limits

- This release verifies local/mock-real HTTP Agent behavior and bad-output
  diagnostics. It does not run paid judge models by default.
- Promptfoo, DeepEval, LangChain AgentEvals, and Ragas remain optional external
  eval integrations unless explicitly enabled by the operator.
