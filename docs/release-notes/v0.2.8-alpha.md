# v0.2.8-alpha

Study Anything v0.2.8-alpha is an API/Skill launch-path release. It removes the rejected standalone
Web UI from the release contract and makes the self-host alpha honest: users run the API, operate it
through CLI/Skill/platform agents, and bring their own reasoning agent.

## Highlights

- Removes the non-shippable React/Vite Web UI from the current launch path.
- Makes the default Docker `core` profile API + app Postgres only.
- Publishes and verifies the GHCR API image without a Web image requirement.
- Adds `GET /v1/sessions/{session_id}/agent-audit` to prove which Agent provider handled the required
  learning tasks.
- Keeps `GET /v1/sessions/{session_id}/agent-eval` as a deprecated one-release alias for invocation
  audit compatibility.
- Updates the mock HTTP Agent smoke so it verifies external Agent coverage for `quiz.generate`,
  `answer.grade`, and `insight.synthesize`.
- Reframes commercial readiness around API/Skill/platform-agent integrations before any new UI work.

## First Launch

Install Docker Desktop or Docker Engine with the Compose plugin, start the daemon, then run:

```bash
python3 scripts/setup_env.py
USE_PUBLISHED_IMAGES=true ./scripts/launch_self_host.sh
```

Open:

- API docs: http://localhost:8000/docs
- API health: http://localhost:8000/v1/health
- System status: http://localhost:8000/v1/system/status

Maintainers can verify the public API image end to end with:

```bash
python3 scripts/verify_published_image_launch.py --tag v0.2.8-alpha
```

## Upgrade

Create a local backup before upgrading:

```bash
python3 scripts/self_host_data.py backup
```

Then pull the release image:

```bash
STUDY_ANYTHING_IMAGE_TAG=v0.2.8-alpha \
USE_PUBLISHED_IMAGES=true \
./scripts/launch_self_host.sh
```

## Known Limitations

- The alpha remains self-host-first. Managed hosting, accounts, billing, Publish, and Teams are not
  included yet.
- There is no shippable standalone Web UI in this release. Future UI work belongs on a separate branch
  after the API/Skill path is stable.
- Agent audit proves invocation coverage, not model quality. Mature quality eval adapters for DeepEval,
  LangChain AgentEvals, and Ragas are planned.
- Study Sync is still a local encrypted package boundary, not hosted remote sync.
- Recovery remains manual. API status surfaces do not execute destructive restore operations.
- Plugin installation remains local and explicit; remote marketplace distribution is not implemented.
- First-run GHCR downloads can take several minutes on slower connections.
