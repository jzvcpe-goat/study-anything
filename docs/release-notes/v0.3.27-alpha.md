# Study Anything v0.3.27-alpha

This alpha adds a release-only cleanroom bootloader. The goal is to let an
external operator or platform Agent start from the GitHub Release page without a
prepared repository checkout, then verify release assets, platform tool imports,
and a minimal runtime path.

## Highlights

- Added `platform/bootstrap/study_anything_release_bootstrap.py`, a standalone
  bootloader that uses Python standard library for metadata-only adoption checks.
- Added `release-cleanroom-bootstrap-v1` runtime reports and
  `release-cleanroom-bootstrap-evidence-v1` generated evidence.
- The bootloader downloads the six public release assets, verifies GitHub
  sha256 digests, unpacks the adoption pack, validates Kimi/Codex/WorkBuddy and
  generic OpenAPI entrypoints, and can delegate runtime replay.
- Runtime modes include metadata-only, Skill Mode, published image, and external
  API. Skill Mode can download the matching tag source archive into a temporary
  directory, so a preexisting checkout is not required.
- Reports include a copyable GitHub issue body and recovery plan while excluding
  raw source text, learner answers, Agent prompts, endpoint secrets, model keys,
  support payloads, and local absolute paths.

## Validation

```bash
python3 platform/bootstrap/study_anything_release_bootstrap.py --tag v0.3.27-alpha --platform kimi --runtime metadata-only
python3 platform/bootstrap/study_anything_release_bootstrap.py --fixture fixtures/release-asset-adoption/asset-only-pass.json --asset-dir platform/generated --runtime skill-mode --source-dir .
python3 scripts/generate_release_cleanroom_bootstrap.py --check
python3 scripts/bootstrap_from_release.py --tag v0.3.27-alpha --runtime metadata-only
python3 scripts/replay_platform_agent_from_release.py --tag v0.3.27-alpha --platform kimi --runtime skill-mode
python3 scripts/generate_platform_adoption_pack.py --check
python3 scripts/verify_ecosystem_submission_pack.py
./scripts/release_check.sh
```

## Known Limits

- Metadata-only mode proves release asset and tool import readiness but does not
  start Study Anything.
- Runtime mode may need network access to download the matching GitHub tag source
  archive unless `--source-dir` is provided.
- Real model choice remains outside Study Anything. The user's platform Agent or
  local Agent gateway owns model credentials, browser access, and external tools.
