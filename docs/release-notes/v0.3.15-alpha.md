# Study Anything v0.3.15-alpha

This alpha hardens Agent eval marketplace enforcement for Kimi, Codex,
WorkBuddy, and generic OpenAPI platform-Agent submissions. It keeps Study
Anything local-first while making external judge usage explicit and testable.

## Added

- `scripts/verify_agent_eval_marketplace_enforcement.py` emits
  `agent-eval-marketplace-enforcement-v1`.
- The verifier checks the external eval runner contract, required-mode failure
  behavior, timeout controls, malformed judge output diagnostics, missing
  Promptfoo runtime diagnostics, baseline regression, platform pack evidence,
  ecosystem submission evidence, and adoption-pack inclusion.
- Promptfoo, DeepEval, LangChain AgentEvals, and Ragas are represented as
  external judge contracts with optional and required execution modes.

## Changed

- Platform packs for Kimi, Codex, and WorkBuddy now require the enforcement
  verifier and acceptance evidence.
- Ecosystem submission, operator drill, manual rehearsal, submission dry-run,
  bundle manifest, adoption pack, and release check now include the enforcement
  gate.
- Docs now distinguish native required eval gates from optional external judge
  adapters that only block release when an operator explicitly uses required
  mode.

## Security And Privacy

- Real model keys and external judge keys remain outside Study Anything.
- Shared eval evidence excludes raw source text, learner answers, Agent endpoint
  secrets, real model credentials, judge credentials, and private browser/video
  context.
- Optional external judge failures are readable diagnostics; required external
  judge failures exit non-zero so marketplace submissions fail closed.

## Verification

```bash
python3 scripts/verify_agent_eval_marketplace_enforcement.py --check
python3 scripts/verify_ecosystem_submission_pack.py
python3 scripts/generate_platform_adoption_pack.py --check
./scripts/release_check.sh
```

## Known Limits

- This release does not add a standalone UI.
- External judge runtimes and judge/model credentials must be installed and
  managed by the operator's platform Agent, CI environment, or local Agent
  workspace.
- LangChain AgentEvals and Ragas remain contract/export targets in this release;
  they are not mandatory runtime dependencies for the local-first core.
