# Study Anything v0.3.31-alpha

Study Anything v0.3.31-alpha is the first Cognitive Loop System release candidate line. It keeps
Study Anything as the learning adapter while adding a local-first cognitive control layer for
project understanding, human gates, evidence bundles, platform Agent adoption, and release
verification.

## Added

- Cognitive Loop contracts under `.cognitive-loop/` for config, permissions, evals, and risk.
- Cognitive Loop CLI/HTML artifact flows for `init`, `verify`, `run-once`, `snapshot`, human gates,
  evidence bundles, event indexes, artifact doctor, repair plans, and recipe execution.
- Platform packs for Kimi-compatible tools, Codex Skill usage, WorkBuddy-style HTTP tools, and
  generic OpenAPI tool import.
- Release stack verifiers for offline readiness, live GitHub checks, full PR lineage to `main`, and
  a maintainer merge runbook.
- GitHub launch operator guide, launch acceptance ledger, adopter evidence archive, release asset
  bootstrap, cleanroom bootstrap, and platform Agent replay evidence.

## Changed

- The public positioning now treats Cognitive Loop System as the main project direction and Study
  Anything as its source-bound learning adapter.
- The launch path is local-first and platform-Agent-first; no standalone frontend is required for
  the current alpha.
- Published-image, release-asset, platform adoption, support, onboarding, and public status evidence
  are bundled into generated adoption packages.
- The default published image tag and release commands now target `v0.3.31-alpha`, preserving the
  already-published `v0.3.30-alpha` prerelease as historical evidence.

## Privacy

- Generated public evidence must not include raw source text, learner answers, Agent endpoint
  secrets, real model keys, judge keys, support bundle private payloads, personal profiles, or
  private browser/video/app context.
- Real model choice, credentials, tools, and external data access stay inside the user's Agent,
  platform Agent, or local gateway. Study Anything validates outputs and records redacted evidence.

## Known Limits

- This release still does not add a standalone UI. The supported adoption path is Skill Mode,
  self-host, and platform Agent/tool import.
- Production Mastra daemon/watch/storage operations remain planned; current Cognitive Loop runtime
  artifacts include local CLI/HTML evidence flows, a repo-local Mastra MVP, local libSQL durable
  proof, and local Langfuse DTO mapping proof.
- Hosted paid services remain not ready before PMF. The commercial path stays future hosted sync,
  teams, trusted ecosystem, and operational convenience.
