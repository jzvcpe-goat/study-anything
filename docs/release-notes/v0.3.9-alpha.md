# v0.3.9-alpha

Study Anything v0.3.9-alpha adds a manual platform-submission rehearsal layer
for Kimi-compatible, Codex Skill, WorkBuddy-style HTTP, and generic OpenAPI
tool integrations.

## What's New

- Adds `platform-manual-submission-rehearsal-v1`, a redacted handoff report for
  external platform operators.
- Verifies the manual acceptance path: unpack the adoption pack, import tools,
  start a local runtime, configure a user-owned HTTP Agent, run a first lesson,
  export learning evidence, and collect diagnostics.
- Adds explicit failure-remediation guidance for tool import, runtime launch,
  Agent health, and evidence redaction.
- Keeps the launch posture unchanged: no standalone frontend requirement, no
  Study Anything custody of real model keys, and no hosted billing in this phase.

## Verification

- `python3 scripts/verify_platform_manual_submission_rehearsal.py --check`
- `python3 scripts/verify_platform_submission_dry_run.py --check`
- `python3 scripts/verify_platform_operator_drill.py --check`
- `python3 scripts/generate_platform_adoption_pack.py --check`
- `python3 scripts/verify_external_adoption.py --pack platform/generated/study-anything-platform-adoption-pack.zip --copy-worktree`
- `./scripts/release_check.sh`

## Known Limits

- This release does not submit to third-party marketplaces automatically.
- Real Agent credentials and model routing remain inside the user's own Agent or
  host platform environment.
- Published-image smoke may report a local GHCR pull timeout; accept that only
  when GitHub `docker-images` succeeds and the GHCR manifest includes
  `linux/amd64` and `linux/arm64`.
