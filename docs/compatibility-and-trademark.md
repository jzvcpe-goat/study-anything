# Compatibility And Trademark

Delivery Clearance and AI Delivery Clearance Protocol identify the public protocol and
reference harness. CBB and `cbb.*` remain technical compatibility identifiers.

Implementations may describe factual interoperability without implying endorsement.

## Permitted Statements

- `CBB Protocol v1 fixture-conformant against <pack digest>`
- `Implements the AI Delivery Clearance Protocol v1 canonical objects`
- `Compatible with delivery-clearance receipts within <named scope>`

Each statement must name the tested implementation version, pack digest, supported
schema set, and claim boundary.

## Prohibited Statements

- `Delivery Clearance certified` without an independent certification program;
- `official`, `approved`, or `endorsed` solely because fixtures pass;
- `AI trusted`, `production safe`, or `compliant` as a consequence of conformance;
- use of the name to conceal incompatible authority expansion;
- use of repository CI as an external audit result.

Forks and independent implementations may use the protocol names descriptively. They
must not impersonate project maintainers, alter receipts while retaining the original
digest, or suggest a commercial or legal guarantee.

## Neutrality

No model provider, Agent platform, cloud vendor, industry, or repository package is
required for protocol conformance. The historical `study-anything` distribution is one
adapter and reference implementation, not the protocol owner in downstream systems.

## Claim Boundary

These rules define accurate compatibility language. They are not trademark registration,
legal advice, certification, or a warranty.
